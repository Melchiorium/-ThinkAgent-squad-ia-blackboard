# Lot 63 - V4 Versioned Documents And Summaries

## Objective

Add V4 helpers for versioned agent documents and controlled LLM summaries.

Documents are the long-form source of truth. Summaries are compressed
derivatives used to reduce context size and must not introduce new information.

## Useful Project Context

- V4 stores documents under:
  - `runs/<run_id>/documents/product/`
  - `runs/<run_id>/documents/tech/`
  - `runs/<run_id>/documents/growth/`
- V4 summaries live under `runs/<run_id>/summaries/`.
- Summaries should be generated by LLM from one source document at a time.
- The summary prompt must be strict and should reject invention.
- V3 prompts must remain untouched. V4 summary prompts should be created under
  `app/prompts V4/`.

## Files Allowed To Modify

- `app/`
- `app/prompts V4/`
- `docs/ai/contracts.yaml`
- `docs/ai/modules.yaml`
- `docs/ai/flows.yaml`
- `docs/ai/rules.yaml`

## Files Not To Modify

- `app/prompts V3/`
- `outputs/tests/`
- `outputs/tests_preliminaires/`
- `outputs/web-jobs/`
- `.env`
- `.env.local`

## Development Steps

1. Read `docs/ai/00-index.yaml`, then memory files related to contracts,
   modules, flows, and rules.
2. Add document helpers, for example in `app/run_documents.py`.
3. Implement helpers to:
   - write an agent document with an explicit name such as `PRD_V0.md`;
   - list documents for an agent;
   - read a named document;
   - read the latest document for an agent.
4. Keep document names explicit and readable. Do not invent a complex version
   resolver.
5. Create `app/prompts V4/summary_prompt.md`.
6. The summary prompt must require this shape:

```yaml
summary:
  source_document:
  source_hash:
  scope:
  key_decisions:
  unresolved_questions:
  critical_risks:
```

7. Add a summary helper, for example `app/run_summaries.py`, that:
   - receives exactly one source document path;
   - computes a source hash;
   - calls the LLM with the strict summary prompt;
   - validates required fields;
   - writes the summary under `runs/<run_id>/summaries/`.
8. If YAML parsing would require a new dependency, use JSON or a simple
   standard-library parser, but keep the persisted file manually readable.
9. Update `docs/ai/contracts.yaml` to describe the V4 document and summary
   contracts.

## Expected Behaviors

- A document can be written, listed, and read without an LLM call.
- A summary is traceable to exactly one source document.
- A stale summary can be detected through `source_hash`.
- The summary prompt explicitly forbids adding information absent from the
  source document.

## Acceptance Criteria

- Versioned document helpers work for Product, Tech, and Growth folders.
- `app/prompts V4/summary_prompt.md` exists.
- A summary includes all required fields.
- Summary generation is source-document scoped.
- Summary validation fails when required fields are missing.
- V3 prompts are unchanged.
- Generated history is untouched.
